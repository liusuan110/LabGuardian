# LabGuardian 计划书改进指南

> **用途**: 供队友参考，基于旧版计划书 (main.pdf) 提出针对性改进建议
> **日期**: 2026-03-03
> **当前系统版本**: v5.1

---

## 一、旧版计划书的主要不足

### 1.1 核心技术深度不够

旧版对系统功能描述较多（"能做什么"），但对核心算法的技术细节不够深入（"怎么做的"）。
评审更关注**你解决了什么难题、用了什么方法、效果如何**。

**需要重点补充的技术点**：

| 技术点 | 旧版状态 | 改进方向 |
|--------|----------|----------|
| 引脚精确定位 | 未提及 | **系统最难的技术点**，需详细描述视觉占用检测 + 几何回退双策略 |
| 面包板网格校准 | 粗略提及 | 4 级检测管线 + Grid RANSAC + 中缝精炼，体现鲁棒性设计 |
| 电路拓扑建模 | 粗略提及 | NetworkX 图 + VF2++ 同构验证 + IC hub 多引脚 |
| 导线端点检测 | 未提及 | 骨架化 (Zhang-Suen) + 颜色分类，纯 CV 无需额外训练 |
| LLM 三级降级 | 笼统描述 | Cloud → Local(NPU) → Rules 的具体策略和降级触发条件 |

### 1.2 缺少实验数据与定量对比

计划书**必须**包含定量实验结果证明方法有效。建议添加：

- 引脚定位精度对比：几何方法 vs 视觉方法 vs 双策略融合
- 假阳性数据：
  - Scene1: 75 → 18 个假占用孔 (降低 76%)
  - Scene3: 214 → 93 (降低 56%)
  - Scene5: 87 → 72 (降低 17%)
- YOLO 检测 mAP：9 类元件分别的 AP 值
- 端到端时延分解：校准 + 检测 + 引脚定位 + 电路建模 + OCR 各步耗时
- 准确率：正确识别元件引脚插入孔位的比率

### 1.3 Intel 平台特色不突出

这是 **Intel Cup**，需要突出 Intel 技术栈的运用，旧版对此着墨过少。

**需要强化的 Intel 元素**：

- **OpenVINO 加速**：YOLO FP16/INT8 量化前后推理速度对比（PyTorch vs OpenVINO）
- **NPU 利用**：Qwen2.5-1.5B INT4 在 NPU 上的推理延迟 vs CPU
- **异构调度设计**：CPU + iGPU + NPU 三单元协同，每个任务分配到最合适的计算单元的理由
- **DK-2500 实测数据**：在竞赛目标平台上的实际性能数据

## 二、计划书结构建议

### 推荐大纲

```
1. 摘要 / 项目概述
   - 一句话定位: "基于 Intel 异构计算的边缘端全离线电子实验智能助教"
   - 核心创新3点

2. 项目背景与需求分析
   - 实验教学痛点 (教师/学生双视角)
   - 同类方案对比 (为什么现有工具不够用)

3. 系统总体设计
   - 四层架构图 ★ (必须有正式配图)
   - AppContext 服务注册中心
   - Image-only 架构决策 (对比视频流方案的理由)

4. 核心技术实现    ← 占比应最大 (40%+)
   4.1 YOLO 元件检测 (HBB+OBB 双模, 9类)
   4.2 面包板网格校准 (4级+RANSAC+中缝精炼)
   4.3 引脚精确定位 ★★ (视觉占用检测 + 几何回退)
       - Median+MAD 鲁棒统计
       - Sigmoid 评分 + 绝对特征门控
       - 自适应搜索半径 + 轴对齐约束
       - 导线阴影聚簇过滤
   4.4 导线骨架分析 (端点+颜色)
   4.5 电路拓扑建模 (NetworkX + IC hub + 4轨电源)
   4.6 电路验证 (VF2++ 图同构)
   4.7 OCR 芯片识别 + IC 引脚数据库
   4.8 LLM 智能问答 (三级降级 + RAG)

5. Intel 异构部署方案 ★
   - CPU / iGPU / NPU 任务分配
   - OpenVINO 量化策略
   - 离线运行保障

6. 实验与评估
   - YOLO 检测精度 (mAP@50, 各类AP)
   - 引脚定位准确率 (对比实验)
   - 端到端延迟分解
   - 在 DK-2500 上的性能数据

7. 教师端 & 课堂模式 (附加功能)

8. 项目进度与分工

9. 总结与展望
```

### 写作风格要点

- **每个技术点**遵循 "WHY → HOW → RESULT" 三段式
  - WHY: 为什么需要解决这个问题
  - HOW: 用了什么方法/算法
  - RESULT: 效果数据
- **多用对比表格**："旧方法 vs 我们的方法"，让评审直观看到改进
- **摘要/引言**开头就点明 "全离线边缘端" 和 "Intel 异构" 两个关键词
- 每到提及计算密集型操作时，主动提 OpenVINO / NPU 加速

## 四、可直接引用的数据与亮点

### 系统规模
- 总代码量: ~11,500 行 / 43 个 Python 源文件
- YOLO 检测类别: 9 类 (电阻/LED/二极管/电容/按键/导线/三极管/IC/变阻器)
- IC 引脚数据库: 3 款 (LM324/LM358/NE5532)
- 面包板校准: 4 级检测管线 + Grid RANSAC
- LLM: 三级降级 (Cloud → Local NPU → Rules)

### 技术创新点 (答辩时可着重讲)
1. **视觉引脚-孔洞占用检测**: 首次将 Median+MAD 鲁棒统计应用于面包板孔洞分析，结合 Sigmoid 评分实现引脚插入位置的精确判定
2. **双策略引脚定位**: 视觉占用检测优先 + 几何候选约束回退，互补提升鲁棒性
3. **导线骨架化端点检测**: Zhang-Suen 骨架化 + 最远点对选择，对弯曲导线天然适配
4. **IC Hub 多引脚拓扑建模**: 将 DIP IC 的 N 个引脚通过中心虚拟节点建模，支持运放等复杂器件
5. **全离线 Intel 异构部署**: YOLO(iGPU) + LLM(NPU) + OCR/CV(CPU) 三单元协同

### 关键实验数据
- 引脚占用检测假阳性降低: 17%~76% (3 个真实场景)
- 面包板孔洞检测: 4 级管线覆盖各种光照/角度
- 支持 1-3 张图片多角度融合分析
- 全管线端到端处理 (含 YOLO + 校准 + 引脚 + 电路建模)
